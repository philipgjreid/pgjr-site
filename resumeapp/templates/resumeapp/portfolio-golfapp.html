Golf Statistics App Project

Project Overview

When I started learning Python I decided to build something that I could use myself. Being a keen golfer I decided that a golf statistics app would be a good idea. So, a golf stats app that I can use on my mobile... how hard could that be?

What I Did

  • Created Requirements - Researched options for how to design and structure the app and tested initial concepts in Excel
  • Developed Backend Functionality Set up models (data tables), views, URL routing, and user authentication using Python and Django
  • Designed and Implemented Frontend - Created responsive layouts with HTML, CSS, and JavaScript and integrated Bootstrap for styling
  • Deployed the Application - Set up version control using Git and managed updates to PythonAnywhere hosted site via GitHub

Skills Used

  • Python - Data manipulation and backend development
  • Django - Web framework and URL routing
  • HTML, CSS, JavaScript - Frontend development and responsive design
  • Bootstrap - Styling and layout design
  • GitHub - Version control

Project Details

Requirements

The starting point, once I had decided to build something using Python, was to sketch out some requirements to help guide the build. This required answering two main questions:

What Statistics to Track?

  • Brainstormed potential statistics to collect, from basic to more advanced. Decided to design something to capture a little more than basic as this would provide some more opportunities to use different techniques.
  • Created mock stats in Excel to refine the data structure. This helped confirm the types of scenarios to be handled, such as where some stats would not be applicable and where handling of null values was going to be required.
  • Researched UI designs for inspiration, adopting ideas for navigation and button-based inputs into my design.

How to Make App Accessible on Any Device?

  • Being new to the concept of python applications, I consulted ChatGPT for mobile app development advice. It was recommended that building a web-based application using either Flask or Django was the best option for someone new to the area.
  • Decided to build towards a web-based app, which led to a few other considerations that I dealt with on my app building journey, including:
    • How to manage the data that my app would be collecting
    • How to host a web applicaton
    • How to manage version control of my app

Development

With some idea of what I wanted to build the next thing was to get into actually building something. I initially used Flask to build a working prototype of my app, although it was missing a lot of things. After researching Django a bit more I decided to transition to use it as the web framework for my app.

Initial Prototype in Flask

  • Built an initial version using Flask that allowed me to get the fundamentals of the app created and running on my localhost.
  • Created functionality to allow a user to go through 18 holes and enter data by clicking buttons, as opposed to selecting from dropdown options, and being able to navigate back and forth to previously entered data.
  • Gained a better understanding of how Python can be used in a practical application.
  • Gained experience in HTML, CSS and Javascript through getting this version to work locally.

Improved Version on Django

  • After a steep learning curve in Flask I decided to look into using Django, which turned out to be a steeper learning curve!
  • After taking some time to learn Django, I benefitted from the built-in user authentication, simple data table creation, and code reusability of Django.
  • My previous experience with data structures and SQL came in handy as Django keeps this stuff in the background, but you still need to know how relational tables work to make sure any foreign keys are set up correctly!

Deployment

When I started out my focus was on Python and getting a practical project under my belt. Little did I know how far into web-development I would end up going (not that far for anyone who actually does this stuff, but a lot farther than I was expecting). So all my work on the app required me to get my head around this stuff so I could deploy my app and actually use it.

Hosting Options

  • Explored hosting options, selecting PythonAnywhere for its beginner-friendly environment (and the free-tier option in case everything went sideways on me).
  • Used some other practice projects and loaded them onto the site to get used to the process and find out the little tips and tricks for getting all the settings to line up.
  • The overall page is now aligned to GitHub to allow me to manage it more effectively.

Version Control

  • Utilized GitHub for version control, learning Git fundamentals to manage the project effectively.
  • Have gained experience across the range of tools and techniques required to get a webpage published.

Final Result

After all that, I now have a mobile-accessible golf stats app that I can use at the golf course. It has a guest login, so feel free to check it out.

Go Have A Look At The Golf App

Login Page

  • A login page that allows users to temporarily login as a guest to have a look at the app.
  • There is a Registration page and associated functionality, but as I am keeping the app to my own use for now I have this functionality turned off.
  • The user authentication aspect is made much easier by Django as it is built-in and requires only minor adjustments to incorporate into a project.

Round Page

  • A page that allows users to enter the details for their round. This includes:
    • The course played, which in my case will normally be Shandon!! I can manually update the app with details for as many courses as I like.
    • The tee, which is based on a subset of tee options for each course.
    • Other details like the date of the round, the type of round, the weather and wind conditions, plus a free-text field for any other notes.
  • There is also a cancel button to allow a user to navigate back to the home screen for the app.
  • This page is based on a Django Form, which takes care of most of the heavy lifting and Bootstrap provides the formatting. The most technical aspect is the script to return the list of tees based on the selected course.

Stats Input Page

This is where the bulk of the work is done. The key features to note are:

  • Hole Details (number and par) - These are provided based on the course being played.
  • Score - This is set to the par of the hole to begin with and buttons are used to increase or decrease the score.
  • Clickable Buttons (Fairwar, Green, Putts, Up/Down, Sand Save, and Penalties) - These all allow one option to be selected, and deselected if necessary. Colour coding of the buttons indicate if the chosen option is good or not.
  • Numeric Options (Drive and First Putt Distance) - Values are manually entered with upper and lower limits for what can be entered.
  • Next / Back Navigation - These are used to move from one hole to another. Moving backwards will repopulate the previously entered data, where available. At the end of the round the 'Next' button turns into 'Complete' to finish the round and move to a summary page.
  • Fairway Values Removed on Par Threes - The two fairway related stats are not applicable for par threes so these get hidden when on a par three.

Future Developments

The main things I have in mind are:

  • Improving the data summary views and analytical capabilities as more data is collected.
  • Consider a more detailed version of collecting stats on a shot-by-shot basis.
