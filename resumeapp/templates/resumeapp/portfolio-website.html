My Personal Website Project

Project Overview

I built a golf stats app to help learn more about Python. Along the way this turned into a web-based app and I decided to extend that project into building myself a personal website.....this website!! Most of what I did is covered in my summary for my golf stats app project (check it out here), but here is a view of what building the website involved.

What I Did

  • Extended Backend and Frontend from Golf Stats App - Took the golf project base and extended to have a full personal website layout
  • Developed and Refined Content Created pages for the various types of content I wanted to include on my site and refined the content and styling
  • Deployed the Application - Set up version control using Git and managed updates to PythonAnywhere hosted site via GitHub

Skills Used

  • Python - Backend development
  • Django - Web framework and URL routing
  • HTML, CSS, JavaScript - Frontend development and responsive design
  • Bootstrap - Styling and layout design
  • GitHub - Version control

Project Details

Extended Site

The golf app had already allowed me to create a Django project with several pages providing the basis to create more pages using similar techniques. The personal website was slightly easier to build due to being mainly static pages. I still needed to work out a few things:

What to Include

  • The inspiration for creating my own site was to be able to represent myself in more detail than what a CV can. This meant that I wanted to start with:
    • CV / Resume - To begin with I decided to recreate my CV in a digital format.
    • Portfolio - Creating a portfolio to exhibit things I have done, either personally or professionally. This obviously included the golf app as well as other projects I have completed and things I continue to work on.

How to Format the Site

  • Site Structure - The overall structure and layout was my first consideration. This largely flowed from the simple content that I was aiming to create. The Resume and Portfolio, plus a Home page...and the golf app of course.
  • Content Formatting - The formatting was the more complicated aspect. To manage things like colour schemes, fonts, and general layout using techniques based in HTML, CSS, and Bootstrap required a lot of trial and error.

Developed Content

With an idea of what I wanted to create I got into making it. Here is an overview of the key things I completed.

Resume Page

  • Resume Project - One of the projects I completed to learn how to use Django was to create a resume site. This used a free HTML template with a lot of really good built-in functionality. I used this as my initial resume page and incorporated it into my overall project.
  • Customisation - The resume template was good, but it came with its own styling and functionality. I customised it to use the styling from my project for consistency. I also replaced the template navbar as I already had a navbar in my overall site.

Portfolio Page and Sub-page Structure

  • The portfolio section needed a landing page and then a way to manage projects and examples as I add to the portfolio.
  • Portfolio Landing Page - When I thought about the types of things I wanted to include in my portfolio, I decided that it fell into either personal projects (like the golf app) or examples from my career. This dictated splitting the landing page into two parts. I did this by:
    • Splitting the page into two sections using a heading that spans the whole page.
    • Using Bootstrap cards and other formatting features to store each of the projects or subjects to make it easy for the content to resize with the window size.
  • Portfolio Pages - Each project or example gets its own page. I have tried to keep the structures of the page fairly simple so they can be reused for multiple subjects without too much readjusting.
    • Similar Bootstrap formatting was used to manage the content in sections that can resize with the size of the screen.

Home Page

  • The concept of a home page got me looking around at overall themes for the site. I came across one that was fairly simple and suited the type of content I was trying to create. I incorporated this into my page by:
    • Copying the example page HTML and CSS into my page.
    • Working out the aspects of the example code that I wanted to keep in terms of functionality and theme.
    • Created my own version of a home page using the techniques I found useful.

General Techniques Used

  • Pre-made Layouts - A helpful tip I got was to use HTML templates to help shortcut creating pages. There was a learning curve for how to use a mix of HTML, CSS and Bootstrap layout and formatting approaches and getting them to work together. While I did copy HTML / CSS from various places to some extent, I tried to manually adjust a lot of it to help learn how things work.
  • AI Support - I leaned heavily on the likes of Copilot and ChatGPT throughout the entire process to find out how to do things, trouble shoot code, and generally work on the components to be able to build the overall site.
  • Online Resources - I spent a lot of time watching instructional videos to learn the fundamentals of things like Django and GitHub and also to help with more specific techniques and approaches.

Deployed the Page

Before deploying my page I needed to make sure that the content was at a level I was happy with and that I was going to be able to get the page hosted and then be able to support it into the future.

Hosting Options

  • Explored hosting options, selecting PythonAnywhere for its beginner-friendly environment.
  • Used some other practice projects and loaded them onto the site to get used to the process and find out the little tips and tricks for getting all the settings to line up.
  • My page is now aligned to GitHub to allow me to manage updates more effectively.

Version Control

  • Utilized GitHub for version control, learning Git fundamentals to manage the project effectively.
  • Have gained experience across the range of tools and techniques required to get a webpage published.

Final Result

I now have a web-page to help promote myself and store my projects. I also learned a lot of different disciplines along the way that will no doubt come in handy in other endeavours.

Future Developments

  • I intend to keep adding things to the page and refining the format and functionality.
