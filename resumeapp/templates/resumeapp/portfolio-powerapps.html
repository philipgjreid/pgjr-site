Power Apps Risk Tool Project

Project Overview

While I was working at the Ministry of Justice (circa 2024) as a risk consultant, I developed a proof-of-concept app using Power Apps for risk and assurance management. The existing risk tool had been created to move from using Excel based risk registers, but still had several limitations and missing functionality. My aim was to show what could be done using Power Apps as a potential future tooling option.

What I Did

  • Current State Assessment - Assessed the current state and documented future state requirements as a baseline for creating the PoC app
  • Developed PoC Risk Tool Developed an app to showcase the potential functionality that could be built using Power Apps
  • Business Case for Risk Tool - Presented the PoC app to my team and incorporated the option into a draft business case

Skills Used

  • Power Apps - App development with DAX and Power Fx
  • Databases - Database structure creation in Dataverse

Project Details

Current State Assessment

The existing solutions had previously moved from using Excel based risk registers to creating a risk tool in Oracle Application Express. While this tool provided a better central repository for managing risks, the solution had several major limitations was relatively simple with a single record for each risk. It tracked risk controls and risk treatments using free-text fields for each risk, which was one of the main drivers to consider a new solution. The main requirements were to create a Control Library and a Treatment Register so those activities could be tracked and linked to multiple risks, where required. An uplift in assurance processes also necessitated the tracking of control assessments, which was going to have to be done using Excel spreadsheets.
