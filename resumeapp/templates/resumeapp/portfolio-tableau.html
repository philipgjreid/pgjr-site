Formula One Tableau Dashboard Project

Project Overview

Objective

To enhance my Tableau knowledge, I created a comprehensive dashboard using Formula One data from ergast.com. The project involved data modelling, dashboard design, and a range of Tableau dashboarding techniques.

What I Did

Data Analysis and Requirement Setting

  • Reviewed the Data - Analysed the dataset to identify valuable insights and define dashboard requirements.
  • Requirements Gathering - Established key requirements based on potential user interests, focusing on historical F1 results, navigation ease, and performance metrics.

Dashboard Design and Layout

  • Dashboard Structure - Outlined the dashboard structure, ensuring it covers races, drivers, and circuits.
  • Layout Planning - Designed a set of dashboards to allow easy navigation of the content and used space-saving techniques like a collapsible filter pane.
  • Theme Implementation - Applied a consistent theme across the dashboards with good use of colour and spacing.

Data Modeling

  • Data Import and Setup - Imported 14 data tables from the ergast dataset into Tableau, maintaining integrity and accuracy.
  • Relationship Management - Adjusted the non-snowflake schema of the ergast dataset to work with the Tableau Logical / Physical data modelling approach.
  • Calculated Measures - Created additional calculated measures to support the required visualisations (e.g., Pole Positions, Race Finish Categories, Flag Images).

Dashboard Development and Refinement

  • Visual Development - Designed and tested a variety of visuals to ensure they accurately displayed the required data.
  • Interactive Elements - Implemented buttons for dashboard navigation and the collapsible filter pane and included filters to enhance user interactivity.
  • Final Formatting - Ensured consistent formatting, optimal usability, and a cohesive look across all dashboards.

Skills Used

  • Tableau - Dashboard creation using Tableau calculations, data modeling, and interactive visuals.
  • Databases - Data modelling within Tableau.
  • Data Analysis - Effective data exploration and requirement gathering.
  • Design and Usability - User-centric design principles, space optimisation, and interactive elements.

Project Details

Data Analysis and Requirement Setting

The normal first step for creating any dashboard or performance reporting content is to start with user requirements. In this case I did not have any users, so I considered the available data to determine the requirements.

Reviewed Existing Data

The data from the ergast dataset extends back to 1950 and has details on races, drivers, and circuits for each F1 season.

  • Amount of Data - The dataset was large and provided a lot of data to consider for inclusion from both the time period covered and the amount of detail available for each race, which extends down to lap times for each driver. Some data points were only available from more recent years, so this also had to be taken into account.
  • Potential Data Views - The dataset provided several different ways to view the data, such as from a specific driver's perspective. This meant a range of views and levels of detail could be considered.

Requirements Gathering

The requirements were based around what someone interested in F1 might like to know. The main requirements are:

  • Overview of F1 History - Provide access to an overview of F1 historical results.
  • Easy Navigation to Points of Interest - Allow users to easily navigate into subject areas of interest.
  • Key Performance Metrics - Provide key metrics, such as championship titles or race wins.

Dashboard Design and Layout

An outline of what the dashboard should include and how it should be structured was created as a starting point.

Dashboard Structure

The available data had details on races, drivers, and circuits, so it made sense to build something that covered all these areas and allow both a high level view as well as more detailed views when required.

  • Landing Page with Multiple Detailed Pages - The structure was based around a landing page that provided an overview of all the available data that could then allow users to navigate to sub-pages that would provide additional details.

Layout Planning

  • Individual Dashboard Designs - Each dashboard was sketched out to consider what content could be included and how they could be organised on the page. The aim was to use Tableau visuals that made sense for the information being provided and ensure each page had a logical layout that was easy to view.
  • Space Saving Techniques - Due to the amount of information to include it was important to maximise the use of space. This was mostly done via the inclusion of a collapsible filter pane.

Theme Implementation

  • Dashboard Theme - The overall theme of the dashboard was kept simple to focus on providing an uncluttered layout with consistent colouring.

Data Modelling

The ergast dataset came with its own data model, so that made the initial set up much easier. The data model consists of 14 tables.

Data Import and Setup

The 14 data tables were imported into Tableau were not all used in the dashboard due to the tables not strictly following a snowflake schema.

  • Model Imported Data - 10 of the 14 data tables were modelled using the Tableau Logical and Physical layer approach. Tableau requires a star or snowflake schema, i.e. start with fact tables and link to dimension tables with downstream tables only having one relationship to a higher table. The F1 dataset did not hold to this, so a partial data model compared to the original was created.
  • Relationship Management - The Tableau Logical layer does not work like traditional models, like what would be found in Power BI. Following the rule of keeping fact tables on the left and dimension tables on the right I was able to create a model that allowed me to create most of the content I required.

Calculated Measures

To support the visuals that needed to be created a range of additional calculated measures were created. The sorts of things created were:

  • Pole Positions - A countnew column in the qualifying table to denote the pole position so this could be easily filtered for in visuals.
  • Race Finish Categories - Several measures to help group race finishes into wins, podiums, top tens, finishes and DNFs. These worked in conjunction with other calculated measures to make categorised counts easy to create.
  • Flag Images - To include flags in the dashboard a calculated measure was created to reference flag images from flagpedia.net.

Dashboard Development and Refinement

The dashboard content was developed and iterated until the technical content was working and then the final formatting was touched up.

Refining Dashboard Content

The visuals for the dashboard all required some form of technical development and testing. With a range of techniques being used, each one needed to be set up and tested and could then be reused as required across the dashboard.

  • Visual Data - Each visual required a certain value or set of values to be displayed. This required a mix of measures and filters to ensure each visual displayed the correct values.
  • Changing Views Buttons - The collapsible filter pane and page navigation were creating using buttons and dashboard layout options.

Final Formatting

The final step was to make small adjustments to the look and feel of the dashboard to optimise usability. This is where having clear requirements and a good initial design makes it easier to assess if the final product is where it needs to be. Typically users or customers would be consulted through the entire design and development process and their feedback would help confirm what good looks like. In my case, it was just me so the final product is simply an example of what can be created.

  • Formatting - Getting all of the visuals to display with consistent fonts, sizing and spacing. This is important to make sure the dashboard is consistent and users can get used to similar design features.
  • Usability - A review of how a user would navigate around and likely use the dashboard to make sure it makes sense and is easy to use. This is where user feedback would allow a better product to be designed.

Final Result

The Formula One Dashboard consists of 5 dashboards to allow users to drill into the data as they see fit.

The best way to see the result is to go and check out the dashboard.

F1 Dashboard on my site

Main Dashboard

The landing page for the dashboard provides:

  • Championship Result Tables - Tables for both the driver and constructor champions for each year.
  • Circuit Map - A map of all the circuits that have been raced at.
  • Driver Grand Prix Appearances - A chart showing the results for all drivers.

Common Features

There are also other features that appear on all dashboards:

  • Dashboard Navigation - There are dashboard navigation buttons in the header bar to allow a user to move around dashboards.
  • Collapsible Filter Pane - The filter pane is on the right side and is collapsed until the filter button is selected and can then be hidden with a close button.

Drivers Dashboard

An overview of all drivers that provides:

  • Driver Nationalities - A bar chart to display the nationalities of all F1 drivers.
  • Driver Longevity - A histogram to show how many races drivers have competed in.
  • Result Tables - Two tables for grand prix results and sprint race results for all drivers. The key metrics are provided and can be used to sort the lists.

Driver Details

A page where one driver can be selected at a time to view their career in more detail.

  • Driver Drop-down Selector - Drop-down selector for the driver that then returns their nationality, age, and the first and most recent years racing in F1.
  • Race History Charts - A race history stacked column chart to show the finishes in each year and a line chart to show the average points and average finish position in each year.
  • Additional Driver Stats - Bar charts to display the average finish at each circuit to show a drivers favourite circuits and a detailed breakdown of reasons for not finishing races.

Constructors Dashboard

An overview of all constructors that provides:

  • Constructor Grand Prix Results - A chart and a table to provide a breakdown of results.
  • Constructor Championships - A treemap or heatmap chart for all constructors based on how many championships they have won.
  • Constructor Nationalities - A bar chart of constructors by nationality.

Circuits Dashboard

An overview of all circuits that provides:

  • Circuit Map - The same map visual from the Main Dashboard as a reference point for the circuits.
  • Fastest Lap Details - A table of the available fastest lap details for each circuit.
  • Circuit Grand Prix History - A char to display which years each circuit was raced at in F1. The tooltip that appears when the chart is hovered over summarises the details and also has a link to the wikipedia page for the circuit.
